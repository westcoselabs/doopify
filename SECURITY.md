# Security Policy

Doopify is currently operated as a private beta.

## Reporting A Vulnerability

If you discover a potential security issue:

1. Do **not** disclose it publicly first.
2. Report it through `/.well-known/security.txt` for this deployment.
3. Include reproduction steps, impact, and any proof-of-concept details needed to validate the issue.

The `/.well-known/security.txt` endpoint is the canonical disclosure channel and returns contact/policy details for the active store configuration.

## Disclosure Expectations

- Please allow maintainers reasonable time to investigate, remediate, and deploy a fix before any public disclosure.
- Avoid sharing exploit details in public issues, discussions, or pull requests.
- If unsure whether behavior is a vulnerability, report it privately first.

## Scope Notes

- This policy covers security issues in the Doopify codebase and deployment behavior.
- Secrets, credentials, or sensitive customer/merchant data must never be posted in public reports.
